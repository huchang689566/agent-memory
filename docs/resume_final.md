### Agent 长短期记忆系统（个人项目）| 2026.01-2026.04

**技术栈：** Python / LangGraph / FAISS / Redis / MCP 协议 / FastAPI / DeepSeek / BGE Embedding

**项目背景：**
LLM Agent 跨会话会丢失用户信息，每次新对话从零开始。已有方案大多把记忆逻辑硬编码在 Agent 内部，导致记忆策略与推理逻辑耦合，扩展困难。

**架构设计：**
三层解耦架构——底层 RAG 记忆服务（FAISS 向量索引）、中间层 MCP 协议网关（标准工具接口）、上层 Agent 推理引擎（MCP 客户端调用记忆服务）。核心原则：Agent 无状态，状态在外部——上下文全部外置到 Redis 和 FAISS，Agent 可随时销毁重建，天然水平扩展。

**项目流程：**
记忆注入：Agent 自主判断新对话，决定是否检索历史记忆注入上下文。记忆提取：LLM 分析对话提取候选 → 向量去重 → 同类别冲突判断（识别重复/新增/更新）→ 落库，冲突更新经审批不静默覆盖。短期记忆：Redis 热数据 + 磁盘双写，滑动窗口按轮裁剪。工具扩展：注册到独立向量库，Agent 语义检索按需发现，新增工具零改动提示词。

**安全设计：**
构建了输入输出纵深防御体系。输入侧：规则引擎检测已知注入模式 → BGE 向量比对识别语义改写 → Wolf-Defender 分类模型（中英文原生支持）兜底隐蔽攻击，三层串联任一命中即拦截，模型不可用时自动回退。输出侧：规则 + 系统提示词向量泄露检测（余弦相似度比对），防止提示词外泄。

**项目业绩：**

- 设计三层解耦架构，记忆存储、工具网关、推理引擎独立部署，解决记忆与推理耦合
- 实现 Agent 无状态范式，进程可随时销毁重建，架构天然支持水平扩展
- 构建长期记忆提取管道和冲突审批机制，保证记忆一致性
- 构建 LLM 输入输出纵深防御体系，规则+向量+分类三层串联，覆盖注入、改写、越狱等场景
- 工具热插拔，新增工具零改动提示词
