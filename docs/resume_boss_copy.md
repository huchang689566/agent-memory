### Agent 长短期记忆系统（个人项目）| 2026.01-2026.04

**技术栈：** Python / LangGraph / FAISS / Redis / MCP / FastAPI / DeepSeek / BGE Embedding

**项目背景：**
LLM Agent 跨会话丢失用户信息，每次新对话从零开始。已有方案把记忆逻辑硬编码在 Agent 内部，导致记忆策略与推理逻辑耦合，扩展困难。

**架构设计：**
三层解耦——底层 RAG 记忆服务（FAISS 向量索引）、中间层 MCP 协议网关（标准工具接口）、上层 Agent 推理引擎（MCP 客户端调用记忆服务）。核心原则：Agent 无状态，状态在外部——上下文全部外置到 Redis 和 FAISS，Agent 可随时销毁重建，靠 user_id + session_id 恢复全部上下文，天然支持水平扩展。

**记忆管道：**
记忆注入：Agent 自主判断新对话，决定是否检索历史记忆注入上下文。记忆提取：LLM 分析对话提取候选 → 向量去重（BGE 相似度比对）→ 同类别冲突判断（识别重复/新增/更新）→ 落库，冲突更新经审批不静默覆盖。短期记忆：Redis 热数据 + 磁盘 JSON 双写，滑动窗口按轮裁剪，保留完整工具调用历史。

**工具系统：**
外部工具注册到独立向量库，Agent 通过语义检索按需发现和加载，新增工具只需定义注册，启动时自动入向量库，提示词零改动。集成了 MCP 多服务协议（Memory MCP + 高德地图 MCP），支持外部工具自动发现和故障回退。

**安全设计：**
构建了输入输出纵深防御体系。输入侧：规则引擎检测已知注入模式 → BGE 向量比对识别语义改写 → Wolf-Defender 分类模型（中英文原生支持）兜底隐蔽攻击，三层串联任一命中即拦截，模型不可用时自动回退。输出侧：规则 + 系统提示词向量泄露检测，余弦相似度阈值比对。各层独立可替换，分类层不加载时不影响主流程。

**业绩：**

- 设计三层解耦架构，记忆存储、工具网关、推理引擎独立部署，解决记忆与推理耦合问题
- 实现 Agent 无状态范式，进程可随时销毁重建不丢失信息，架构天然水平扩展
- 构建 LLM 输入输出纵深防御体系，规则+向量+分类三层串联，覆盖注入、改写、越狱，各层独立可替换
- 长期记忆提取管道和冲突审批机制，保证记忆一致性

### 企业知识采集与问答双机器人系统 | 2026.04-2026.06

**技术栈：** Dify / 通义千问 / 多模态视觉模型 / LangBot / 企业微信 / 混合检索 + Reranker

**项目背景：**
企业 IT 运维中客服每天处理大量重复问题，但解决方案散落在聊天记录无法复用。需要自动化的知识沉淀和消费闭环。

**架构设计：**
基于 Dify 构建知识采集和知识问答两个独立工作流，通过 LangBot 长连接对接企业微信智能机器人。两个工作流通过 Dify 知识库 API 共享数据，采集机器人的输出直接成为问答机器人的检索源，形成知识闭环。

**项目流程：**
采集流程：用户在企微发送客服聊天截图 → 多模态视觉模型识别对话中的系统问题、根因分析和解决方案 → 自动推断所属业务系统 → 输出结构化 Q&A 对。提取结果先缓存在对话变量中，用户审核确认后通过 API 写入 Dify 知识库。入库前自动进行信息脱敏处理（身份证号、手机号、工号等），支持追加修改后重新提交确认。

问答流程：用户提问 → 问题分类器意图识别 5 分支路由。当用户提问模糊（未指明具体系统）时，反问用户澄清系统名称，将补充信息与原始问题拼接后进行二次检索，实现多轮对话中的需求澄清。多轮上下文通过 Dify 对话变量缓存（上一轮系统名和问题），回答完成后自动清理，不依赖外部存储。检索策略以采集机器人沉淀的 Q&A 知识库为主、企业系统操作文档为辅，双路混合检索加重排序，LLM 基于检索上下文生成回答并自动拼接去重后的引用来源，禁止编造。

**业绩：**

- 采集→消费闭环架构，两个工作流通过 API 解耦共享数据，消除人工中转和维护成本
- 多模态模型自动将截图转化为结构化知识，入库前自动脱敏，保证信息安全合规
- 对话变量管理多轮上下文，轻量无需外部存储，检索命中率和回答准确率显著提升
- 多分支意图分类覆盖模糊咨询场景，回答禁止编造、来源可追溯

### 企业文档 RAG 检索增强系统 | 2025.01-2025.10

**技术栈：** Python / mineru / Milvus / BGE Embedding / BM25 + 向量混合检索 / BGE-Reranker / 通义千问 / FastAPI

**项目背景：**
企业内部存在大量格式混乱的文档——PDF 扫描件、DOCX 操作手册、PPTX 培训材料等，格式不统一导致传统检索方案需要大量人工整理。同时 RAG 系统的效果评估缺乏标准方法：纯粹人工评估成本高，纯粹自动评估又无法判断检索结果和回答的真实质量。

**架构设计：**
构建了数据清洗 → 检索增强 → 效果评测三层 RAG 管线。清洗层将多格式文档统一转化为结构化文本块，检索层负责混合检索和基于上下文的答案生成，评测层建立三层体系做持续质量监控。三层各自独立迭代，清洗层输出作为检索层的输入，检索层日志作为评测层的数据来源。

**数据清洗：**
mineru 统一处理 PDF/DOCX/PPTX/扫描件等异构格式，利用视觉分析自动区分标题、正文、图片和表格区域，不依赖文档格式标注。block 级输出展平后按语义分块——计算相邻句子的向量相似度，在相似度骤降处切开，分块粒度由语义边界决定而非固定字符数。章节拆分采用双策略：有标题结构的文档按标题层级预切后语义合并，无标题结构的文档直接全篇分块，同一套流程兼容两类文档。图片通过 OCR 提取文字替代，分块时通过句子到页码的映射关系追踪每个 chunk 在原始文档中的位置。

**检索增强：**
采用 BM25 关键词检索 + BGE 向量语义检索双路召回，关键词匹配保证精确命中，向量语义保证泛化覆盖，召回结果经 BGE-Reranker 精排后取 top-K。LLM 选型考虑指令跟随能力而非推理能力——要求模型严格基于检索上下文生成回答，不添加上下文之外的信息，因此选用通义千问而非推理型模型。回答附带来源文档名和页码范围，方便用户回溯原文核查。

**评测体系：**
建立三层人工+自动结合的评测机制。第一层离线回归——维护标注测试集，每次修改检索策略后自动跑关键词覆盖回归，纯向量计算无需 LLM 参与，保证改动不引入回归。第二层线上自动信号——监控空结果率、用户点踩率、重复提问率三个指标，超标自动标记触发人工审查。第三层人工抽查——定期从线上日志随机抽样，逐条验证检索质量和大模型回答正确性，对问题做归因分析和策略纠偏。

**安全考量：**
识别了检索文档内容拼入 prompt 时的间接注入风险——恶意文档可嵌入指令操控模型输出。设计了文档内容预检 + LLM 输出校验双层过滤方案作为后续迭代方向。

**业绩：**

- 语义分块替代固定长度分块，解决异构文档格式导致的检索质量不一致问题
- BM25+向量双路混合检索 + Reranker 精排，关键词和语义互补减少单一方式的漏召回
- LLM 选型侧重指令跟随而非推理能力，有效减少检索上下文不足时的编造问题
- 建立三层评测体系，明确 RAG 评测中自动化与人工的合理分工边界——自动信号缩小排查范围，人工抽查精准判断质量，不做虚假全自动评测
- 识别间接注入攻击面并设计防御方案，完善了 RAG 安全防护体系
